부록 G. 용어 빠른 풀이 — 처음 만나는 영어·전문 용어 한 줄 사전

이 책은 코봇(Cobot)과 플로봇(Flowbot)이라는 친근한 비유로 출발하지만, Copilot Studio가 실제로 쓰는 용어 가운데에는 영어 이름이나 기술 용어가 적지 않다. 이 부록은 본문을 읽다가 “이게 무슨 말이지?” 싶을 때 즉시 찾아볼 수 있도록, 일반 독자(비개발 직장인) 눈높이의 한 줄 풀이를 모은 것이다.

읽는 법 본문은 각 용어가 처음 나오는 장에서 짧게 풀이한다. 더 자세한 설명이 필요하거나 잊었을 때 이 표로 돌아오면 된다. 제품 기능·이름은 2026년 6월(미리보기 포함) 기준이며, 최신 사항은 컴패니언 사이트에서 확인한다.

가장 자주 나오는 6개 (먼저 외워 두면 편한 닻 용어)

용어 한 줄 풀이
오케스트레이션 (orchestration) 코봇이 지침·지식·도구를 언제 무엇을 꺼내 쓸지 스스로 판단하는 조율 엔진. 오케스트라의 지휘자에 해당한다.
Dataverse Power Platform이 데이터를 보관·관리하는 회사 공용 데이터 금고. 고객 목록·프로젝트 대장 같은 구조화 정보가 여기 담긴다.
DLP (Data Loss Prevention) 회사 자료가 허락 없이 빠져나가지 못하게 막는 보안 ‘교통 규칙’. 어떤 시스템끼리 데이터를 주고받아도 되는지 정한다.
MCP (Model Context Protocol) 여러 도구와 자료를 ‘도구 상자째’ 코봇에 연결하는 표준 방식.
ALM (Application Lifecycle Management) 코봇을 개발→테스트→운영 환경으로 승격하며 팀 자산으로 관리하는 체계.
BYOM (Bring Your Own Model) 외부(Azure AI 등)에서 직접 가져온 내 AI 모델을 코봇에 끼워 쓰는 방식. 별도 과금 대상이다.

처음 나오는 장 순서로 보는 전체 사전

용어 처음 나오는 장 한 줄 풀이
LLM / 거대언어모델 1장 ChatGPT 같은, 말을 이해하고 생성하는 AI 언어 모델.
오케스트레이션 (orchestration) 1장 코봇이 지침·지식·도구를 언제 꺼낼지 자동 판단하는 조율 엔진(지휘자).
Skill 1장 필요할 때 꺼내 쓰는 ‘전문 업무 모자’ = 업무별 매뉴얼.
컨텍스트 불안 (context anxiety) 2장 작업 기억이 한계에 가까워지면 모델이 조급해져 일관성을 잃는 현상.
one-shot 소진 2장 한 번에 다 처리하려다 작업 기억(컨텍스트)을 중간에 소진해 일이 끊기는 실패.
코딩 하네스 (coding harness) 2장 LLM을 실행 루프로 감싸 도구를 직접 쓰게 하는 런타임. ‘말에 씌우는 마구(馬具)’에 빗댄 말.
CLI 2장 명령줄로 도구를 부리는 터미널형 조작 방식.
long-horizon 작업 2장 여러 단계를 끝까지 이어가야 하는 긴 작업.
opt-in 2장 강제가 아니라 필요할 때 스스로 선택해 들어가는 방식.
Workflows / 플로봇 3장 정해진 절차를 어김없이 도는 자동화 설비. 옛 ‘agent flows’와는 다른 새 도구.
결정론적 (deterministic) 3장 같은 입력이면 항상 같은 순서·결과가 나오는 성질.
Components 5장 Build 탭에서 코봇에 붙이는 능력 조각(모델·지식·도구 등).
tray / surface (트레이·표면) 5장 화면 영역을 가리키는 UI 번역어.
Dataverse 6장 Power Platform의 회사 공용 데이터 금고.
DLP (Data Loss Prevention) 6장 자료가 허락 없이 빠져나가지 못하게 막는 보안 교통 규칙.
Schema name 6장 시스템이 코봇을 부를 때 쓰는 고유 이름(표시 이름과 별개, 첫 저장 후 고정).
ALM 6장 코봇을 개발→테스트→운영으로 승격·관리하는 체계.
환경 변수 / dev·prod / 솔루션 6장 환경별로 달라지는 값 / 연습·실제 환경 분리 / 환경 사이로 옮기는 묶음 꾸러미.
Deep reasoning 7장 복잡한 문제를 단계별로 깊게 추론하는 모델 능력(느리고 비용 높음).
BYOM (Bring Your Own Model) 7장 외부에서 가져온 내 AI 모델(별도 과금).
Preview / GA / Experimental 7장 모델·기능 상태 표시: 미리보기 / 정식 출시 / 실험.
단일 출처 (single source of truth) 8장 같은 규칙을 여러 곳에 베끼지 않고 한 곳에만 두는 설계 원칙.
on-demand 로드 9장 필요한 순간에 딱 꺼내 쓰는 방식.
description (설명문) 9장 언제 이 Skill·도구를 쓰는지 코봇에게 알려주는 호출 설명서.
마크다운 (Markdown) 9장 복잡한 꾸미기 없이 글만으로 문서 구조를 잡는 간단한 작성 양식.
컨텍스트 (context) 9장 코봇이 한 번에 펼쳐 놓고 볼 수 있는 정보량(작업대 크기).
보안 트리밍 (security trimming) 10장 내가 볼 권한이 없는 자료는 코봇도 보지 못하게 잘라 내는 것.
MCP (Model Context Protocol) 10장 여러 도구·자료를 도구 상자째 연결하는 표준 방식.
REST API 11장 다른 시스템에 일을 시키는 표준 연결 통로.
OpenAPI / YAML 11장 어떤 명령을 어떻게 보내는지 적어 둔 API 사용 설명서 파일.
커넥터 (connector) 11장 Outlook·Teams 같은 서비스에 바로 붙는, 준비된 손발.
Computer use 11장 사람처럼 화면을 클릭·입력해 조작하는 미리보기 기능(신중히 사용).
Connected agents 12장 다른 전문가 에이전트를 팀원처럼 불러 쓰는 기능(옛 child agent를 대체).
슈퍼호스트 12장 혼자 일하던 코봇이 팀장처럼 다른 전문가를 지휘하는 단계.
agent node 14장 플로봇(절차) 한가운데에서 코봇(판단)을 불러내는 블록.
Classify / Extract 14장 텍스트를 범주로 분류 / 문서에서 원하는 값만 뽑아내는 AI 액션.
핸드오프 (handoff) 14장 코봇과 플로봇이 일의 운전대를 서로 넘겨주는 설계 지점.
에이전트 호출 트리거 (When an agent calls the flow) 15장 코봇이 플로봇을 부르는 신호(트리거).
chain of thought 17장 코봇이 답에 이르기까지 거친 추론 과정.
tool call 17장 코봇이 도구를 실제로 불러 행동한 기록(말과 행동의 차이를 보여 줌).
Pass rate 18장 합격률 — 테스트 묶음 중 통과한 비율.
Regression (회귀) 18장 무언가를 고쳤더니 잘 되던 기능이 다시 깨지는 현상.
Copilot Credits 21장 코봇이 일한 양을 재는 사용량 눈금(대화·응답·작업 실행에서 소비).
Entra ID 21장 Microsoft 조직 로그인 체계(옛 이름 Azure AD).
