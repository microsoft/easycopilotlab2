부록 E. 능력 카탈로그

Skill, Knowledge, Tool을 “무엇을 붙일 수 있는가” 관점에서 모은 살아있는 참조 부록. 본문 9~11장은 원리를 설명하고, 이 부록은 목록과 선택 기준을 제공한다.
기준: Microsoft Learn 확인, 2026년 6월. 미리보기 기능과 한도는 변경될 수 있다.
읽을 사람: “여기에 무엇을 더 붙일 수 있나”를 통째로 훑고 싶은 사람.

E4. Knowledge sources 요약

Microsoft Learn의 Knowledge sources summary 기준으로, Copilot Studio는 에이전트가 Power Platform, Dynamics 365, 웹사이트, 외부 시스템의 데이터를 근거로 답하게 할 수 있다. 내부 지식은 사용자의 Microsoft Entra ID 권한을 따르며, 코봇이 권한을 우회하지 않는다.

지식 소스 내부/외부 무엇을 하나 2026년 기준 generative mode 입력 한도 인증
Public website 외부 지정 웹사이트를 Bing으로 검색 최대 25개 웹사이트 없음
Documents 내부 Dataverse에 업로드한 문서 검색 모든 문서 없음/환경 권한
SharePoint 내부 SharePoint URL을 Graph Search로 검색 최대 25개 URL 사용자 Entra ID
Dataverse 내부 Dataverse 테이블을 RAG 방식으로 검색 제한 없음 사용자 Entra ID
Enterprise data using connectors 내부 Microsoft Search로 색인된 커넥터 데이터 검색 제한 없음 사용자 Entra ID

Knowledge 운영 규칙

규칙 설명
보안 트리밍 질문한 사용자가 볼 수 있는 내부 자료만 답변에 사용된다.
읽기 전용 Knowledge는 참고·인용용이다. 수정·업로드·발송은 Tool이다.
설명 품질 지식 소스가 많으면 설명(description)이 선택 품질에 영향을 준다.
25개 초과 필터링 generative orchestration에서 지식 소스가 25개를 넘으면 내부 모델이 설명을 바탕으로 관련 소스를 필터링한다. 업로드 파일은 이 25개 검색 제한에 포함되지 않는다고 Learn은 설명한다.
Web Search 별도 설정으로 공개 웹 검색을 사용할 수 있다. 조직 정책과 데이터 경계를 확인한다.
Work IQ SharePoint 등 조직 데이터 검색 품질을 높일 수 있다. M365 Copilot 라이선스·Semantic index·Microsoft 인증 조건을 확인한다.

파일 크기 관련 확인 포인트

항목 확인된 기준
Work IQ 사용 시 Learn은 SharePoint와 커넥터 파일 최대 200MB 지원을 설명한다.
특정 파일 형식 SharePoint와 Microsoft Copilot connectors는 PDF/PPTX/DOCX에 대해 최대 512MB 지원 설명이 있다.
운영 적용 테넌트·라이선스·파일 형식·검색 인덱스에 따라 달라질 수 있으므로 게시 전 실제 환경에서 확인한다.

E6. Tool 유형 카탈로그

새 경험의 Tool은 코봇이 실제 행동을 하게 하는 손발이다. Learn의 새 경험 개요는 Tool을 API 호출이나 flow 실행처럼 “일을 수행하는 능력”으로 설명한다. 아래 목록은 2026년 기준 실무에서 자주 만나는 유형이다.

Tool 유형 무엇을 하나 예 주의
Prebuilt connector Microsoft/타사 앱의 기성 작업 실행 Outlook, Teams, SharePoint, Planner DLP와 사용자 권한 확인
Custom connector 사내 API를 커넥터로 감쌈 ERP, 그룹웨어 개발·인증·버전 관리 필요
REST API API 엔드포인트 직접 호출 외부 SaaS, 내부 서비스 입력 검증·오류 처리 필수
Workflow as tool 코봇이 결정적 절차를 호출 승인, 등록, 발송 실행 액션 용량 소비
MCP 표준 프로토콜로 도구 묶음 연결 사내 도구 서버 도구 설명·스키마 품질 중요
Computer use API 없는 UI를 조작 레거시 앱 미리보기/정책/감사 확인
M365 Copilot/Agent node(Workflow 안) Workflow가 M365 Copilot이나 에이전트를 호출 분석·조사·예외 판단 새 Workflows 설계 범위에서 다룸
Classify/Extract(Workflow 안) 분류·추출 인텔리전트 액션 티켓 분류, 필드 추출 M365 Copilot node와 구분

Prompt 주의
클래식/이전 문서에서 보던 Prompt 기능을 신버전 에이전트의 현재 설계 요소처럼 쓰지 않는다. 신버전에서는 Agent 쪽은 Skill, Workflow 쪽은 agent node로 설명한다.

E8. Connected agents 카탈로그 예시

전문가 에이전트 언제 분화하나 코봇의 역할
예산 전문가 예산 데이터·승인 도구·재무팀 소유가 분리될 때 요청을 요약해 넘기고 결과를 종합
리스크 전문가 규정·컴플라이언스 지식이 크고 독립 평가가 필요할 때 위험 판단을 받아 프로젝트 계획에 반영
고객지원 전문가 티켓 시스템·응대 지식·운영 지표가 별도일 때 사용자 대화의 단일 창구 유지
법무 검토 전문가 법무팀 소유·검토 기준이 명확할 때 자문이 아니라 검토 요청과 결과 정리

Connected agents는 슈퍼호스트 패턴의 핵심이다. 다만 많이 연결할수록 좋은 것이 아니다. 소유권과 평가 기준이 분명한 전문가만 연결한다.

항목 원칙
제품 명칭 Learn의 새 경험 문서 기준으로 갱신
미리보기 “2026년 기준 미리보기, 변경 가능”을 붙인다
한도 숫자는 공식 문서 확인 시점과 함께 쓴다
내부 FAQ 대외비 문장 인용 금지, 사실만 공개 표현으로 재서술
본문 연결 Skill=9장, 지식/기억=10장, 도구=11장, 슈퍼호스트=12장, Workflows/트리거=4부

이 카탈로그는 닫힌 목록이 아니라 자라는 목록이다. 새 경험은 빠르게 넓어지고 있고, 갱신할 때 다음 갈래를 함께 채워 넣는다 — 스킬을 가져다 쓰는 마켓플레이스, 사람이 직접 드나드는 앱(창구), 큰 파일을 통째로 다루는 파일 처리, 소비를 한눈에 보는 비용 대시보드, 그리고 스스로 기억하고 도는 자율 동료다. 각 항목의 정확한 명칭·가용성·한도는 빠르게 바뀌므로, 이 표는 “읽는 법”으로 쓰고 최신 목록은 화면과 컴패니언 사이트에서 확인한다.
